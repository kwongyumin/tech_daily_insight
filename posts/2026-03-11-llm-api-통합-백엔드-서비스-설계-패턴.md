# LLM API 통합 백엔드 서비스 설계 패턴

## 개요

ChatGPT, Claude, Gemini 등 대형 언어 모델(LLM) API가 실무 서비스에 빠르게 스며들고 있다. 단순한 프로토타입 수준을 넘어, 수십만 사용자가 사용하는 프로덕션 환경에서 LLM을 안정적으로 운영하려면 일반적인 REST API 연동과는 다른 접근이 필요하다.

LLM API는 다음과 같은 고유한 특성을 가진다.

- **높은 레이턴시**: 응답 생성까지 수 초에서 수십 초가 소요된다.
- **비결정론적 응답**: 동일 입력에도 매번 다른 결과가 나온다.
- **토큰 기반 과금**: 요청/응답 토큰 수에 따라 비용이 결정된다.
- **Rate Limit**: 분당 요청 수, 토큰 수에 엄격한 제한이 있다.
- **서비스 가용성 불확실성**: 외부 서비스이므로 장애나 지연이 언제든 발생할 수 있다.

이 글에서는 이러한 특성을 고려한 백엔드 서비스 설계 패턴을 Spring Boot 기반으로 설명한다.

---

## 핵심 개념

### 1. Provider Abstraction Layer (공급자 추상화)

특정 LLM 벤더에 코드가 종속되면 마이그레이션 비용이 급격히 증가한다. OpenAI에서 Claude로, 혹은 사내 오픈소스 모델로 전환할 때 비즈니스 로직 변경 없이 교체 가능한 구조가 필요하다.

### 2. Streaming 응답 처리

LLM의 응답은 SSE(Server-Sent Events) 또는 WebSocket을 통해 스트리밍으로 전달하면 UX를 크게 개선할 수 있다. 백엔드는 LLM의 스트림을 받아 클라이언트로 그대로 전달하는 파이프라인 역할을 한다.

### 3. Fallback & Multi-Provider 전략

하나의 LLM 서비스에만 의존하면 해당 서비스 장애 시 전체 기능이 마비된다. Primary/Secondary 모델을 지정하고 자동 전환하는 Fallback 메커니즘이 필수다.

### 4. Prompt 관리와 버전 관리

하드코딩된 프롬프트는 운영 중에 수정할 수 없다. 프롬프트를 별도로 관리하고 A/B 테스트, 버전 롤백이 가능한 구조가 실무에서 필수적이다.

OpenAI 구현체:

```java
@Component
@RequiredArgsConstructor
public class OpenAiProvider implements LlmProvider {

    private final OpenAiClient openAiClient; // WebClient 기반 HTTP 클라이언트
    private final OpenAiProperties properties;

    @Override
    public String getProviderName() {
        return "openai";
    }

    @Override
    public LlmResponse complete(LlmRequest request) {
        OpenAiChatRequest apiRequest = buildRequest(request);
        return openAiClient.chat(apiRequest)
                .map(this::mapToLlmResponse)
                .block(Duration.ofSeconds(60));
    }

    @Override
    public Flux<String> streamComplete(LlmRequest request) {
        OpenAiChatRequest apiRequest = buildStreamRequest(request);
        return openAiClient.streamChat(apiRequest)
                .map(chunk -> chunk.getChoices().get(0).getDelta().getContent())
                .filter(content -> content != null && !content.isEmpty());
    }

    private OpenAiChatRequest buildRequest(LlmRequest request) {
        List<OpenAiMessage> messages = new ArrayList<>();
        if (request.getSystemPrompt() != null) {
            messages.add(new OpenAiMessage("system", request.getSystemPrompt()));
        }
        messages.add(new OpenAiMessage("user", request.getUserMessage()));

        return OpenAiChatRequest.builder()
                .model(properties.getDefaultModel())
                .messages(messages)
                .temperature(request.getOptions().getTemperature())
                .maxTokens(request.getOptions().getMaxTokens())
                .build();
    }
}
```

### Fallback 전략 구현

```java
@Service
@RequiredArgsConstructor
@Slf4j
public class LlmService {

    private final List<LlmProvider> providers;
    private final ProviderHealthChecker healthChecker;
    private final MeterRegistry meterRegistry;

    public LlmResponse completeWithFallback(LlmRequest request) {
        List<LlmProvider> healthyProviders = providers.stream()
                .filter(p -> healthChecker.isHealthy(p.getProviderName()))
                .collect(Collectors.toList());

        for (LlmProvider provider : healthyProviders) {
            try {
                Timer.Sample sample = Timer.start(meterRegistry);
                LlmResponse response = provider.complete(request);
                sample.stop(meterRegistry.timer("llm.request",
                        "provider", provider.getProviderName(),
                        "status", "success"));
                return response;
            } catch (LlmRateLimitException e) {
                log.warn("Rate limit exceeded for provider: {}", provider.getProviderName());
                healthChecker.markDegraded(provider.getProviderName(), Duration.ofMinutes(1));
            } catch (LlmProviderException e) {
                log.error("Provider {} failed: {}", provider.getProviderName(), e.getMessage());
                healthChecker.markUnhealthy(provider.getProviderName());
            }
        }
        throw new AllProvidersFailedException("All LLM providers are unavailable");
    }
}
```

### Streaming SSE 엔드포인트

```java
@RestController
@RequestMapping("/api/v1/llm")
@RequiredArgsConstructor
public class LlmController {

    private final LlmService llmService;
    private final PromptTemplateService promptTemplateService;

    @PostMapping(value = "/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<ServerSentEvent<String>> streamChat(
            @RequestBody ChatRequest chatRequest,
            @AuthenticationPrincipal UserDetails user) {

        String resolvedPrompt = promptTemplateService.resolve(
                chatRequest.getTemplateId(),
                chatRequest.getVariables()
        );

        LlmRequest llmRequest = LlmRequest.builder()
                .systemPrompt(resolvedPrompt)
                .userMessage(chatRequest.getMessage())
                .options(LlmOptions.builder()
                        .maxTokens(1024)
                        .temperature(0.7)
                        .build())
                .build();

        return llmService.streamWithFallback(llmRequest)
                .map(token -> ServerSentEvent.<String>builder()
                        .data(token)
                        .build())
                .concatWith(Flux.just(ServerSentEvent.<String>builder()
                        .event("done")
                        .data("[DONE]")
                        .build()))
                .onErrorResume(e -> Flux.just(ServerSentEvent.<String>builder()
                        .event("error")
                        .data(e.getMessage())
                        .build()));
    }
}
```

### 프롬프트 버전 관리

프롬프트를 DB 또는 설정 저장소에서 관리한다.

```java
@Entity
@Table(name = "prompt_templates")
@Getter
@Builder
public class PromptTemplate {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String templateKey;     // 예: "customer-support-v2"
    private Integer version;
    private String content;         // 실제 프롬프트 내용 ({{variable}} 형태 포함)
    private boolean active;

    @CreationTimestamp
    private LocalDateTime createdAt;
}

@Service
@RequiredArgsConstructor
@Slf4j
public class PromptTemplateService {

    private final PromptTemplateRepository repository;
    private final LoadingCache<String, PromptTemplate> cache;

    public String resolve(String templateKey, Map<String, String> variables) {
        PromptTemplate template = cache.get(templateKey);
        String content = template.getContent();

        for (Map.Entry<String, String> entry : variables.entrySet()) {
            content = content.replace("{{" + entry.getKey() + "}}", entry.getValue());
        }
        return content;
    }

    // Caffeine Cache 설정
    @Bean
    public LoadingCache<String, PromptTemplate> promptCache(PromptTemplateRepository repository) {
        return Caffeine.newBuilder()
                .expireAfterWrite(5, TimeUnit.MINUTES)
                .maximumSize(100)
                .build(key -> repository.findByTemplateKeyAndActiveTrue(key)
                        .orElseThrow(() -> new PromptNotFoundException(key)));
    }
}
```

### Token 사용량 추적 및 비용 관리

```java
@Component
@RequiredArgsConstructor
public class TokenUsageTracker {

    private final TokenUsageRepository repository;
    private final ApplicationEventPublisher eventPublisher;

    @Async
    public void track(String userId, String provider, TokenUsage usage) {
        TokenUsageRecord record = TokenUsageRecord.builder()
                .userId(userId)
                .provider(provider)
                .promptTokens(usage.getPromptTokens())
                .completionTokens(usage.getCompletionTokens())
                .totalTokens(usage.getTotalTokens())
                .estimatedCostUsd(calculateCost(provider, usage))
                .recordedAt(Instant.now())
                .build();

        repository.save(record);

        // 임계치 초과 시 알림 이벤트 발행
        checkBudgetThreshold(userId, record);
    }

    private double calculateCost(String provider, TokenUsage usage) {
        // 공급자별 토큰 단가 적용
        PricingPolicy policy = PricingRegistry.get(provider);
        return (usage.getPromptTokens() * policy.getInputPricePerToken())
             + (usage.getCompletionTokens() * policy.getOutputPricePerToken());
    }
}
```

## 주의사항 및 트레이드오프

### 타임아웃 설정의 딜레마

LLM 응답은 최대 30~60초까지 걸릴 수 있다. API Gateway나 로드 밸런서의 기본 타임아웃(30초)과 충돌할 수 있으므로, **인프라 전 계층의 타임아웃 값을 일관되게 설정**해야 한다. Streaming을 사용하면 첫 토큰 도착 시점에 연결이 유지되어 타임아웃을 실질적으로 우회할 수 있다.

### 동기 vs 비동기 처리

응답 시간이 긴 요청을 동기로 처리하면 스레드 풀이 빠르게 고갈된다. **WebFlux 기반의 비동기 파이프라인** 또는 **작업 큐(Kafka/SQS) + 폴링 방식**을 검토해야 한다. 사용자가 즉각적인 응답을 기다리는 인터랙티브 서비스라면 Streaming SSE가, 배치 처리나 백그라운드 작업이라면 비동기 큐 방식이 적합하다.

### 캐싱의 한계

비결정론적 특성 때문에 단순 캐싱이 어렵지만, **의미론적 캐싱(Semantic Caching)**을 사용하면 유사한 질문에 대해 이전 응답을 재활용할 수 있다. 다만 RAG(Retrieval-Augmented Generation) 기반 서비스처럼 최신 정보가 중요한 경우에는 캐시 TTL을 매우 짧게 가져가거나 캐싱을 포기해야 한다.

### 프롬프트 인젝션 방어

사용자 입력이 프롬프트에 직접 삽입되는 경우, 악의적인 사용자가 시스템 프롬프트를 무력화하거나 민감한 정보를 추출하려 시도할 수 있다. 입력값 검증, 역할 분리(system/user 메시지 구분), 출력 필터링을 반드시 구현해야 한다.

### 비용 폭발 방지

토큰 제한 없이 운영하면 비용이 예상치 못하게 급증할 수 있다. 사용자별/조직별 **일일 토큰 예산 제한**과 **요청 수 Rate Limiting**을 서비스 레이어에서 강제하는 것이 필수다.
